# OPTRION Product Roadmap

This document outlines the planned trajectory for OPTRION, an Engineering Intelligence Platform. It details upcoming features, architecture changes, and milestones.

---

## v1.1 — Ecosystem Integrations & Infrastructure Extensions

Focus: Expanding the platform's collection inputs and notification outputs to fit standard cloud-native environments.

### Health Collection
- **[ ] Prometheus & OpenTelemetry Collector:** Native ingestion of metrics scraped from Prometheus scrape endpoints and OTLP logs/metrics streams.
- **[ ] Kubernetes Collector:** Automated auto-discovery of Kubernetes pods, services, and ingress objects, monitoring container restarts and status.
- **[ ] TCP & DNS Collectors:** Support for lower-level probe requests to verify network-level infrastructure health.

### Alerting & Notifications
- **[ ] Slack & Discord Webhooks:** Native notification channel adapters for enterprise team channels.
- **[ ] PagerDuty & Opsgenie integration:** Webhook and API mappings to support paging alerts for on-call rotation schedules.

---

## v1.2 — Analytical Observability & SLIs/SLOs

Focus: Elevating platform metrics into higher-level Service Level Indicators (SLIs) and Service Level Objectives (SLOs).

### Dashboards & Visualization
- **[ ] SLO & Error Budget Tracking:** Dashboard visualizations to define and monitor quarterly error budgets (e.g., 99.9% availability).
- **[ ] Interactive Dependency Graph:** Enhanced SVG/canvas service topology map supporting drill-downs into downstream system impacts.

### Intelligence Layer
- **[ ] Statistical Anomaly Tuning:** Custom threshold tuning for Z-Score anomaly detectors on a per-service basis.
- **[ ] Historic Baseline Comparisons:** Materialized views comparing current telemetry against weekly and monthly rolling averages.

---

## v2.0 — Distributed Agent Architecture & Advanced Diagnostics

Focus: Shifting from a centralized pull model to a hybrid push/pull model with edge check execution and deeper AI timelines.

### Platform Scaling
- **[ ] Distributed Check Agents (Probes):** Edge worker daemons that execute health checks locally in different regions and report back to the core Monolith via gRPC.
- **[ ] Database Partitioning Enforcement:** Time-series partitioning of telemetry metrics tables using PostgreSQL pg_partman to handle billions of rows.

### AI Diagnostics (War Room)
- **[ ] Multi-Service Root Cause Analysis (RCA):** Deeper LLM analysis using service dependency maps to trace how an outage in Database X caused anomalies in Gateway Y.
- **[ ] AI-Generated Incident Timelines:** Autogenerated post-mortem summaries detailing exactly when health started degrading, what anomalies fired, and how it resolved.

---

*Note: As an open-source platform, community feedback heavily influences our roadmap. Feel free to open an issue or contribute to shape these goals!*
